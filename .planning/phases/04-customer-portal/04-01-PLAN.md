Create a new migration file at `mounttrack/supabase/migrations/0003_portal_token.sql` with the following exact content:

```sql
-- Migration: add portal_token to jobs table
-- Each job gets a unique UUID token used as the credential for the customer portal.
-- gen_random_uuid() is built-in to PostgreSQL 13+ (pgcrypto extension not required).

ALTER TABLE jobs
  ADD COLUMN IF NOT EXISTS portal_token UUID NOT NULL DEFAULT gen_random_uuid() UNIQUE;

-- Index for fast token lookups on every portal page load
CREATE INDEX IF NOT EXISTS jobs_portal_token_idx ON jobs (portal_token);

-- Backfill any existing rows that somehow got NULL (shouldn't happen with NOT NULL DEFAULT,
-- but guards against edge cases in partial migrations)
UPDATE jobs SET portal_token = gen_random_uuid() WHERE portal_token IS NULL;
```

Apply the migration by running the Supabase CLI:
```
cd mounttrack && npx supabase db push
```

If `supabase db push` fails because the remote DB is not linked, run instead:
```
cd mounttrack && npx supabase migration up --local
```

If both fail (no local Supabase running), create the file only — the executor will note in the SUMMARY that the migration must be applied before Plan 02 executes. Do NOT skip creating the file.

The `Database.jobs` Insert and Update shapes inherit from Job via Omit, so `portal_token` will automatically appear in Row. However, `portal_token` has a DB DEFAULT (gen_random_uuid()), so it should be optional on Insert. Update the Insert shape to make it optional:

```typescript
Insert: Omit<Job, 'id' | 'created_at' | 'updated_at' | 'job_number' | 'is_overdue' | 'portal_token'> & {
  id?: string
  job_number?: number
  portal_token?: string
}
```

Leave Update unchanged (Partial<Omit<...>> already makes everything optional).

**Part B — Update service.ts:**

Add `{ auth: { persistSession: false } }` as the third argument to createClient to prevent session caching in server-only portal renders:

```typescript
import { createClient } from '@supabase/supabase-js'
import type { Database } from '@/types/database'

export function createServiceClient() {
  return createClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!,

  )
}
```
